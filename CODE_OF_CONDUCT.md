# Code of Conduct

The **Internets** project adopts the **Contributor Covenant, version 2.1**
as its baseline code of conduct. The canonical text lives at:

<https://www.contributor-covenant.org/version/2/1/code_of_conduct/>

Rather than reproduce that text here, this file summarises how it is
applied to this project in practice.

## Expected behaviour

Be constructive, be respectful, and stay focused on the work. Assume the
person on the other side of an issue or pull request is trying to make
the bot better, and respond in kind. Disagreements about technical
choices are expected and welcome; please keep them about the choice, not
the person making it.

## Unacceptable conduct

Behaviour that targets, demeans, or excludes other contributors is not
welcome in this project's spaces.

## Enforcement

For ordinary disagreements or community-level concerns, the
[GitHub Discussions tab](https://github.com/brandontroidl/Internets/discussions)
is the right venue.

For serious or sensitive concerns — anything you would not be
comfortable raising in public — use the same private channel as
[`SECURITY.md`](./SECURITY.md): GitHub Private Vulnerability Reporting
on the repository's Security tab, or the maintainer profile linked from
`pyproject.toml`'s Homepage. Reports will be handled discreetly.

## Scope

This code of conduct applies to all project spaces — issues, pull
requests, Discussions, commit and review comments, and any other channel
operated under this repository — and to anyone representing the project
in public forums.
